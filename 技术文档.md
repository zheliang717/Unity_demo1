# 画线救狗头 — 技术文档

## 一、游戏概述

一款 2D 画线保护类小游戏。玩家通过手指/鼠标绘制线条，阻挡蜜蜂攻击狗头。蜜蜂使用 A* 寻路算法自动追踪狗头，玩家需要在关键通道处画线拦截。坚持 10 秒即通关。

## 三、核心模块详解

### 3.1 GameManager — 游戏状态管理

- **单例模式**：全局唯一实例，跨场景保留
- **状态机**：`Waiting → Playing → Failed/Won`
- **倒计时**：固定 10 秒，归零触发胜利
- **事件系统**：`OnStateChanged` 通知所有订阅者

### 3.2 LevelManager — 关卡地形

- 3 个关卡，纯代码生成墙体（BoxCollider2D + SpriteRenderer）
- 关卡编号通过静态变量 `s_nextLevel` 跨场景重载持久化
- 关卡设计：
  - **第 1 关**：锯齿通道，5 道横墙交替留缺口
  - **第 2 关**：走廊迷宫，竖墙+横墙组合出房间和瓶颈
  - **第 3 关**：蛇形迷宫，7 层横墙交错+5 道竖墙+底部收窄

### 3.3 Bee — 蜜蜂 AI

- **A* 寻路**：读取 LevelManager 的障碍物数据构建 0.4 单位精度导航网格
- **路径跟随**：沿 A* 路径点移动，每 0.5 秒重新计算（狗头可能被推动）
- **碰撞处理**：
  - 碰到狗头 → 推开狗头 + 游戏失败
  - 碰到线条 → 施加推力 + 清空路径重新寻路
  - 碰到地形 → 沿墙滑行
- **物理参数**：质量 1.5，零弹性材质，drag=3 减少惯性

### 3.4 LineDrawer — 画线系统

- **绘制流程**：
  1. `Begin()`：创建容器（LineRenderer + Rigidbody2D Kinematic）+ 第一个碰撞段
  2. `Move()`：实时添加点和碰撞段，更新 LineRenderer
  3. `End()`：RDP 简化路径 → 重建碰撞段 → Rigidbody2D 切 Dynamic 受重力
- **复合碰撞体**：容器为刚体，子物体 BoxCollider2D 无刚体，Unity2D 自动合并
- **useWorldSpace=false**：LineRenderer 用本地坐标，跟随容器下落

### 3.5 BeePathfinder — A* 寻路

- **网格构建**：读取 `LevelManager.terrainWalls` 的 bounds，膨胀 0.3 单位
- **A* 算法**：8 方向移动（含对角线），曼哈顿距离启发
- **路径平滑**：射线检测删除可直达的中间路径点
- **容错**：起点/终点在墙内时自动找最近可通行格子

### 3.6 PhysicsInit — 碰撞矩阵

运行时强制设置 2D 碰撞矩阵（先全部禁用，再按需开启）：

| | Line | Bee | Dog | Boundary |
|---|---|---|---|---|
| **Line** | - | ✓ | ✓ | ✓ |
| **Bee** | ✓ | ✗ | ✓ | ✓ |
| **Dog** | ✓ | ✓ | - | ✓ |
| **Boundary** | ✓ | ✓ | ✓ | - |

### 3.7 UIManager — 界面

- **HUD**：倒计时数字（红色+正弦摇晃+最后 3 秒脉冲）+ 关卡名
- **结果弹窗**：通关/失败提示 + 下一关/重试按钮
- 使用 IMGUI 方案，全比例布局适配不同屏幕

## 四、物理层级

| 层级名 | 用途 |
|---|---|
| Line | 玩家绘制的线条 |
| Bee | 蜜蜂 |
| Dog | 狗头（保护目标） |
| Boundary | 地形墙体 + 屏幕边界 |

## 五、运行方式

1. 用 Tuanjie/Unity 打开项目
2. 打开 `Assets/Scenes/MainScene.unity`
3. 点击 Play 运行
4. 按住鼠标/触摸屏幕画线，松手后线条下落

---

## 六、技术栈

- **引擎**：Tuanjie 1.9.1（Unity 2022.3 分支）
- **物理**：Unity 2D Physics（Rigidbody2D + BoxCollider2D）
- **渲染**：LineRenderer + SpriteRenderer
- **UI**：IMGUI（OnGUI）
- **寻路**：自实现 A* 算法（网格+8方向+路径平滑）
- **算法**：Ramer-Douglas-Peucker 路径简化
